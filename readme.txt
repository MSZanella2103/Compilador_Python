/*======Como executar o programa*/
O programa possui 2 arquivos "Analise_lexica.py" e o "Analise_lexica&sintatica.py", como o próprio nome subentendisse o Analise_lexica.py mostrará apenas a Analise léxica do código fornecido, já o Analise_lexica&sintatica.py fará toda a parte léxica e sintática do código fornecido.
O código a ser analisado precisa estar dentro do arquivo "exemplo.py", o programa fará a leitura do arquivo e mostrará o código a partir dele

/*OBRIGATÓRIO*/
Para executar o código sem erros precisa intalar 2 bibliotecas
ply -> pip install ply
tabulate -> pip install tabulate


/*======Regras Linguagem Pascal Simplificada*/

Tokens básicos: A linguagem Pascal simplificada é composta por tokens básicos, como identificadores, palavras-chave, números, símbolos especiais e comentários.

Identificadores: Os identificadores são sequências de letras e dígitos, começando com uma letra.
Exemplos de identificadores válidos são "x", "variavel1", "contador", etc.

Palavras-chave: A linguagem Pascal simplificada possui um conjunto limitado de palavras-chave reservadas,como "program", "var", "begin", "end", "if", "then", "else", "while",
"do", entre outras.

Números: Os números podem ser inteiros ou reais. Os inteiros são sequências de dígitos, enquanto os reais possuem uma parte inteira seguida de uma parte decimal opcional,
separada por um ponto. Exemplos de números válidos são "123", "3.14", "0.5", etc.

Símbolos especiais: A linguagem Pascal simplificada utiliza símbolos especiais, como operadores aritméticos (+, -, *, /), operadores de atribuição (:=),
delimitadores de blocos (",", ".", ";", "(", ")"), operadores de comparação (=, <>, <, >, <=, >=), entre outros.

Comentários: Os comentários podem ser incluídos no código fonte para fornecer informações adicionais e são
ignorados pelo analisador léxico. Em Pascal simplificado, os comentários podem ser iniciados por "{", seguidos de qualquer texto, e terminados com "}".
